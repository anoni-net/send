## Change Log

The entries below v4.0.0 are Mozilla-era, generated by the upstream release
tooling, and stop long before upstream's own v3.4.27. They are kept as history.

### v5.1.0 (2026/07/21)

37 commits since v5.0.0. This release works through a pre-release audit of the
whole app: the user-facing failure modes it found, the accessibility gaps, the
dependency backlog, and the supply chain. No change to the wire protocol, the
URL format, or the encryption, so existing links and `ffsend` keep working.
`/__version__` reports `release: v5.1.0`, `version` stays `v3.4.27`.

#### Fixed, user-facing

- **A blocked WebSocket connection hung the upload forever.** `new WebSocket()`
  does not throw on a refused or filtered connection, so the promise never
  settled and the tile sat at 0% with a Cancel that did nothing. It now fails,
  cancels, or times out, and says which. This is the difference between "your
  network is blocking this" and an app that looks frozen.
- **Rate limiting had no UI.** A 429 on upload or download surfaced as a generic
  "something went wrong" page whose only button invited an immediate retry that
  burned another slot. It now explains the wait and reads the server's
  `Retry-After`. The streaming path also stopped masking every error as a silent
  reload.
- **"This link has expired" was shown for a file that had hit its download
  limit.** The server cannot tell the two apart once the record is gone, so the
  message no longer claims a cause it does not know.
- **Unsupported-browser pages pointed at Firefox**, contradicting the footer's
  own "not affiliated with Mozilla or Firefox". They are browser-neutral now,
  and an instance served over plain HTTP explains that the encryption APIs need a
  secure origin rather than blaming the visitor's browser.
- Copy-link no longer reports success it did not check; the JavaScript-disabled
  page is styled again (its classes were being purged); several dialog and copy
  interactions were fixed.

#### Accessibility

Dialogs are real dialogs (`role`, focus trap, Escape, focus restore); the
wrong-password message announces itself; the expiry and download-count dropdowns
have accessible names.

#### Smaller and more current

The application bundle is now 53 KB gzipped, down from 71 KB, after dropping the
`core-js` polyfills a WebCrypto-capable browser never uses.

- `@babel` 7 → 8, `tailwindcss` 2 → 3 (built-in JIT replaces the separate
  purgecss step), `stylelint` 14 → 17, `@fluent/*`, `prettier` 1 → 3, and the
  css/postcss loader chain. Each dependency that changes `dist/` was verified
  with a real browser and, for the CSS, a 20-shot pixel comparison.
- `ws` 7 → 8 under express-ws, verified through a real WebSocket round-trip.
- The production dependency tree keeps shrinking: `cldr-core` (1.2 MB) is down to
  the one 52 KB file the app reads, and nanohtml's Babel-transform dependencies
  (including `acorn`) are stripped from the image since server-side rendering
  never loads them. `npm audit --omit=dev` reports zero.

#### Localization

- The five fork-added footer strings, previously English-only outside zh-TW and
  zh-CN, now cover 33 major languages. Locales without them fall back to English.
- `te` and `ast` dropped from the served set: below 70% coverage, they showed the
  home page and download explanation in English.
- A node locale check runs in CI (replacing an unmaintained Python tool that had
  not run in a long time), so a broken `.ftl` or a served locale with no
  directory fails the build.

#### Supply chain

- `SHA256SUMS.txt` is itself signed now, so the list that catches a tampered
  server is not protected only by a release ACL.
- Trivy scans a saved tarball instead of a socket-mounted `:latest` image; every
  GitHub Action and the base image are pinned by digest, with Dependabot moving
  the pins.
- Reproducibility is a CI gate: two Node versions build the same commit and the
  hashes are diffed, rather than the claim resting on a one-time manual check.
- The image that is tested, scanned and checksummed is now the exact image that
  is signed: the build is pushed by digest, pulled back, and everything runs
  against that.

### v5.0.0 (2026/07/20)

41 commits since v4.0.0. The theme is a smaller trust surface: fewer packages
running in your browser, fewer ways for the server to misbehave, and more of
the claims in this repository checkable rather than asserted.

Findings from the pre-release audit that were deliberately not fixed here are
tracked in [#79](https://github.com/anoni-net/send/issues/79).

#### Compatibility

Still no change to the wire protocol, the URL format, or the encryption.
Existing share links keep working, and so do third-party clients such as
`ffsend`. `/__version__` continues to report `version: v3.4.27` for the API
generation and `release` for ours, now `v5.0.0`.

#### Security

Six defects found by auditing the server rather than by anyone hitting them.

- **Concurrent downloads could exceed the download limit.** The count was read,
  compared, then incremented, so simultaneous requests for a one-download file
  all saw the same pre-increment value. Reserving a slot is now a single atomic
  step (redis `EVAL`, `EXISTS` guarding `HINCRBY`), which also stops a release
  racing behind a delete from resurrecting a spent id as a TTL-less key.
- **Expired files were unreachable but never deleted.** redis holds the only
  expiry TTL; when it fired, the metadata vanished and the file it pointed at
  stayed on disk. A periodic sweep now removes files whose record is gone
  (filesystem backend; S3 and GCS use bucket lifecycle rules).
- **A crafted WebSocket frame crashed the process.** `ws` emits `'error'` for
  anything malformed on the wire, and an `'error'` with no listener is rethrown.
  No authentication required. Encrypted metadata was also unbounded, so one
  request could write 100 MiB into redis; it is now capped at 64 KiB.
- **No rate limiting anywhere.** Per-IP limits now cover the HTTP API and,
  separately and more tightly, the WebSocket upload. `TRUST_PROXY` makes the
  proxy depth explicit, since the limit is only correct when `req.ip` is.
- **The rate limiter could be switched off by one request header.** The skip for
  WebSocket upgrades tested only for the presence of `Upgrade`, so any request
  carrying that header was served without being counted.
- **Upload parameters were not type-checked.** Every comparison against a
  non-numeric value is false, so `timeLimit: "abc"` passed validation, produced
  the filename prefix `NaN`, and yielded a file that no TTL covered and no sweep
  would ever match. Values are coerced and range-checked, and the storage layer
  refuses a non-finite expiry rather than encoding it into a filename.

Removed: the unauthenticated legacy HTTP `/api/upload` route, and a WebSocket
fallback address pointing at Mozilla's shut-down service.

#### What runs in your browser

- **`choo` and its ecosystem are gone**, replaced by a small hand-written
  framework and emitter. Fourteen packages fewer reach the browser.
  `nanohtml` and `nanomorph` are kept deliberately: the first is the escaping
  boundary, the second preserves focus in password fields.
- **All telemetry and tracking removed**, front and back. Nothing this service
  runs reports to anyone else. The CSP enforces it rather than promising it.
- **`helmet` 3 -> 8, with the CSP written out in full** rather than merged with
  library defaults, adding `base-uri`, `script-src-attr`, `form-action` and
  `object-src`. For a service whose security rests on the served JavaScript
  being the published JavaScript, that policy should be readable in one place.
- The main bundle is 70 KB gzipped.

#### Server and build

- `express` 4 -> 5, `mozlog` replaced by a 50-line logger (it reached us through
  `intel`, which calls a function removed in Node 23 and blocked moving off
  Node 22).
- S3 and GCS SDKs are now optional. They were more than half the production
  dependency tree for code a filesystem deployment never loads. 134 packages
  ship, and `npm audit --omit=dev` reports zero vulnerabilities.
- The published image sets `NODE_ENV=production`. It previously inherited the
  `development` default, which switches off both the CSP and the rate limits, so
  a container built from this repository served neither unless its operator
  knew to pass the variable.
- 22 deferred lint warnings cleared; the ceiling is now zero.

#### Verifiability

- `SHA256SUMS.txt` is published with each release, and `VERIFYING.md` documents
  three independent ways to check what an instance served you.
- The build is reproducible. A clean clone plus `npm ci && npm run build`
  produces `dist/` byte-identical to the published image, verified across macOS
  with Node 26 and Linux with Node 22, all 136 files.
- `release` is now read from `package.json` alone. It used to fall back from
  `CIRCLE_TAG`, a dead CircleCI leftover; reaching for the GitHub equivalent
  would have made `dist/` depend on the build environment and broken the
  reproducibility guarantee. A publish-time check requires the git tag to agree
  with the tree.
- **Lint and both test suites now gate the release.** They ran on pull requests
  but matched neither tag pushes nor `main`, so the artifact that gets signed
  and attested was the one artifact they never ran against.
- The smoke test asserts the CSP headers and the rate limit against every image
  build, alongside the existing real `ffsend` encrypted round-trip.

#### Localization and docs

- zh-TW described end-to-end encryption as 點對點加密, the standard rendering of
  peer-to-peer. Corrected to 端對端加密 on the two screens that exist to explain
  the guarantee.
- Documentation inherited from upstream was corrected against the code, and two
  Mozilla-era UX specs describing a Firefox Accounts product that no longer
  exists were deleted. README now carries a Traditional Chinese version.

### v4.0.0 (2026/07/18)

First release under anoni.net's own identity, built on upstream
[timvisee/send](https://gitlab.com/timvisee/send) v3.4.27. 67 commits.

Our version numbers diverge from upstream's here. See "Versioning" below for
what `/__version__` reports and why.

#### Compatibility

No change to the wire protocol, the URL format, or the encryption. Existing
share links keep working, and so do third-party clients such as `ffsend`.
`app/keychain.js`, `app/ece.js`, `app/api.js`, the upload/download/metadata
routes and the HMAC auth middleware have no commits since v3.4.27.

#### Versioning

`/__version__` now reports two fields:

- `version` stays `v3.4.27`. Clients gate on it to pick an API generation:
  `ffsend` maps `>=3.0 <4.0` to its V3 API and has no fallback, so advertising
  `4.x` there breaks every CLI user with "unsupported version".
- `release` is ours, `v4.0.0`. Sentry's release tag reads this field.

#### Runtime

- Node 16 -> 22 (`node:16.13-alpine3.13` -> `node:22-alpine3.22`, both stages).
- `redis` 3.1.1 -> 6.0.0. Storage rewritten for the promise API and the
  camelCase commands; `redis-mock` dropped for an in-memory fake.
- `aws-sdk` 2.1109.0 -> `@aws-sdk/client-s3` + `@aws-sdk/lib-storage` 3.700.0.
- `@google-cloud/storage` 6.2.3 -> 7.21.0.
- `@sentry/browser` 5.30.0 -> 7.120.3.
- `@dannycoates/express-ws` -> mainline `express-ws` 5.0.2, which drops the
  unmaintained `esm` loader that crashes on Node 22.

#### Build

- webpack 4 -> 5, including asset modules in place of `file-loader`,
  `raw-loader`, `html-loader`, `expose-loader`, `extract-loader` and
  `script-loader`, and `xxhash64` hashing so `--openssl-legacy-provider` is no
  longer needed.
- `webpack-dev-server` -> v5, `webpack-dev-middleware` -> v7.
- ESLint 6 -> 10 with a flat config replacing seven `.eslintrc` files.

#### Fixed

- Blank page: webpack 5 defaults `output.publicPath` to `auto`, which prefixed
  every asset URL and 404'd the bundle.
- Missing logo: SVGs through `svgo-loader` export CommonJS while images export
  ESM, so a `.default`-only asset map produced `src="undefined"`.
- Missing "plus" icon in the upload UI.
- Cancelling a download reported an error. Sentry v7 instruments `fetch`, so an
  aborted transfer surfaced as `TypeError` instead of the shape the handler
  matched on. Cancellation intent is now recorded rather than inferred.

#### Security and supply chain

- 46 of 47 vulnerability alerts resolved: 1 critical, 16 high, 19 medium, 10
  low. The one left open is `uuid` (medium), held back by `teeny-request ^9`.
- Images are built and published by our CI to `ghcr.io/anoni-net/send`, signed
  with cosign (keyless, Sigstore), with an SBOM and SLSA provenance attached,
  and scanned with Trivy.
- Multi-arch `linux/amd64` + `linux/arm64`, each built and tested on a native
  runner.
- `npm overrides` pin seven transitive packages.

#### Tests and CI

- Backend: 56 tests, including the first coverage GCS storage has ever had.
- Frontend: 21 tests, moved from puppeteer 2 to Playwright, running in CI.
- Smoke test on every build: health endpoints, a `/__version__` assertion, page
  render with the exact asset URLs the page references, a real `ffsend`
  encrypted upload and download round-trip, and a headless browser check.
- Lint now runs in CI, with a ceiling so its warning backlog cannot grow.
